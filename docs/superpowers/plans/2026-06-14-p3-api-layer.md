# P3 — Next.js API Layer — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Invoke the project `nextjs-patterns` skill before writing any Route Handler.

**Goal:** Build the complete Next.js 16 App Router Route Handler API layer that the UI phases (P4–P6) consume: shared TypeScript types, the server-only Pub/Sub publisher and GCS read helpers, timezone-aware formatting, and every Route Handler under `app/api/**` — projects (list/create with immediate-refresh + backfill fan-out, get/patch/delete), weather blob serving from the private bucket, snapshots, NWAC, SNOTEL, mountains list, the `mountains/[slug]` get-or-refresh browse path, and the admin trigger-refresh endpoint. All handlers unit-tested with mocked `@/lib/firebase-admin` + `@/lib/pubsub`, plus emulator-backed integration tests. Vitest coverage gates lines/functions ≥90, branches ≥85.

**Architecture:** Route Handlers live under `app/api/**` (App Router convention; contract §1). They read Firestore via the Admin SDK singleton `lib/firebase-admin.ts` (created in P0), read combined forecast blobs from the **private** weather bucket via `lib/storage.ts`, and publish Pub/Sub messages via `lib/pubsub.ts`. No external weather APIs are called from the API layer — only Firestore, GCS, and Pub/Sub. The Python workers (P1/P2) are the writers; the API layer is read + publish-trigger only, except for project create/patch/delete which write the `projects` doc. `BROWSE_REFRESH_MODE` (contract §2) gates the lazy get-or-refresh path in `mountains/[slug]`.

**Tech Stack:** Next.js 16.2.x (App Router, React 19.2, TypeScript), `firebase-admin`, `@google-cloud/pubsub`, `@google-cloud/storage`, Vitest 2.x + Testing Library, Firebase emulator suite (Firestore + Pub/Sub). Reuse from P0: `lib/env.ts` (`requireEnv`, `getBrowseRefreshMode`) and `lib/firebase-admin.ts` (`getDb`). **Do not recreate them.**

**Next.js 16 conventions** (apply in every handler):
- Dynamic `params` is a **Promise** — `export async function GET(req: Request, { params }: { params: Promise<{ id: string }> }) { const { id } = await params; … }` (contract §7; nextjs-patterns skill).
- GET Route Handlers are **uncached by default** — every GET sets `Cache-Control: public, max-age=300, stale-while-revalidate=600` explicitly on the response (contract §7).
- Use `NextResponse.json(body, { status, headers })`. Return typed error responses (`{ error: string }` + status), do not throw from handlers.
- `serverExternalPackages` already lists `firebase-admin`, `@google-cloud/pubsub`, `@google-cloud/storage` (P0 `next.config.ts`).

**References:** `docs/superpowers/specs/2026-06-14-mountain-weather-poc-design.md` (spec — esp. §4 browse get-or-refresh + `BROWSE_REFRESH_MODE`, §5 creation flow) and `docs/superpowers/specs/2026-06-14-interface-contract.md` (contract — the source of truth). Section numbers below (e.g. "contract §7") refer to the contract. Skills: `nextjs-patterns` (Route Handler conventions). P0 plan `docs/superpowers/plans/2026-06-14-p0-foundation.md` is the format exemplar and created the reused `lib/env.ts` + `lib/firebase-admin.ts`.

```ts
import "server-only";
import { PubSub } from "@google-cloud/pubsub";
import { requireEnv } from "@/lib/env";

export type LogicalTopic = "weather-refresh" | "backfill-refresh" | "nwac-refresh" | "snotel-refresh";

const ENV_VAR: Record<LogicalTopic, string> = {
  "weather-refresh": "TOPIC_WEATHER_REFRESH",
  "backfill-refresh": "TOPIC_BACKFILL_REFRESH",
  "nwac-refresh": "TOPIC_NWAC_REFRESH",
  "snotel-refresh": "TOPIC_SNOTEL_REFRESH",
};

export interface WeatherRefreshMessage { mountainId: string; reason: "scheduled" | "on_create" | "manual" }
export interface BackfillRefreshMessage { projectId: string; mountainId: string; targetDate: string }
export interface NwacRefreshMessage { zoneId: string }
export interface SnotelRefreshMessage { stationId: string }

let client: PubSub | undefined;
function getClient(): PubSub {
  if (!client) client = new PubSub({ projectId: requireEnv("GCP_PROJECT") });
  return client;
}

export async function publish(logicalTopic: LogicalTopic, message: object): Promise<string> {
  const topicPath = requireEnv(ENV_VAR[logicalTopic]);
  const data = Buffer.from(JSON.stringify(message), "utf8");
  return getClient().topic(topicPath).publishMessage({ data });
}
```

> `import "server-only"` guards against accidental client-component import. The package `server-only` ships with Next.js; if Vitest cannot resolve it in the jsdom env, alias it to a no-op in `vitest.config.ts` (`resolve.alias`: `"server-only": fileURLToPath(new URL("./test/server-only-stub.ts", import.meta.url))` with `export {}`). Add the stub only if the test fails to resolve the import.

```ts
export interface FakeDoc { id: string; exists: boolean; data: () => unknown }
export function fakeDoc(id: string, data: unknown | undefined): FakeDoc {
  return { id, exists: data !== undefined, data: () => data };
}

/** Build a chainable Firestore mock. `seed` maps "collection/doc" -> data and
 *  "collection" -> array of {id,...data} for list queries. */
export function makeDb(seed: {
  docs?: Record<string, unknown>;
  collections?: Record<string, Array<{ id: string } & Record<string, unknown>>>;
}) {
  const docs = seed.docs ?? {};
  const collections = seed.collections ?? {};
  const sets: Array<{ path: string; data: unknown }> = [];
  const updates: Array<{ path: string; data: unknown }> = [];

  function collection(name: string) {
    const list = collections[name] ?? [];
    const query = {
      _items: list.slice(),
      orderBy(field: string, dir: "asc" | "desc" = "asc") {
        this._items.sort((a, b) => {
          const av = String(a[field] ?? ""); const bv = String(b[field] ?? "");
          return dir === "desc" ? bv.localeCompare(av) : av.localeCompare(bv);
        });
        return this;
      },
      where() { return this; },
      limit(n: number) { this._items = this._items.slice(0, n); return this; },
      async get() {
        return { empty: this._items.length === 0, docs: this._items.map((it) => fakeDoc(it.id, it)) };
      },
    };
    return {
      ...query,
      doc(id: string) {
        const path = `${name}/${id}`;
        return {
          id,
          async get() { return fakeDoc(id, docs[path]); },
          async set(data: unknown) { sets.push({ path, data }); docs[path] = data; },
          async update(data: unknown) { updates.push({ path, data }); },
          collection(sub: string) { return collection(`${name}/${id}/${sub}`); },
        };
      },
      async add(data: Record<string, unknown>) {
        const id = `gen-${Math.random().toString(36).slice(2, 8)}`;
        sets.push({ path: `${name}/${id}`, data });
        docs[`${name}/${id}`] = data;
        return { id };
      },
    };
  }

  return {
    db: { collection },
    _sets: sets,
    _updates: updates,
  };
}

export function makePublish() {
  const calls: Array<{ topic: string; message: unknown }> = [];
  const publish = vi.fn(async (topic: string, message: unknown) => { calls.push({ topic, message }); return "msg-id"; });
  return { publish, calls };
```

```ts
import { NextResponse } from "next/server";
import { getDb } from "@/lib/firebase-admin";
import { publish } from "@/lib/pubsub";
import type { Project } from "@/lib/types";

const CACHE = "public, max-age=300, stale-while-revalidate=600";

export async function GET() {
  const snap = await getDb().collection("projects").orderBy("createdAt", "desc").get();
  const projects = snap.docs.map((d) => ({ id: d.id, ...(d.data() as object) }));
  return NextResponse.json(projects, { headers: { "Cache-Control": CACHE } });
}

interface CreateBody {
  name: string; mountainId: string; targetDateStart: string; targetDateEnd: string; notes?: string;
}

function validate(body: unknown): CreateBody | null {
  if (typeof body !== "object" || body === null) return null;
  const b = body as Record<string, unknown>;
  const ok =
    typeof b.name === "string" && b.name.trim() !== "" &&
    typeof b.mountainId === "string" && b.mountainId !== "" &&
    typeof b.targetDateStart === "string" && b.targetDateStart !== "" &&
    typeof b.targetDateEnd === "string" && b.targetDateEnd !== "";
  if (!ok) return null;
  return {
    name: (b.name as string).trim(), mountainId: b.mountainId as string,
    targetDateStart: b.targetDateStart as string, targetDateEnd: b.targetDateEnd as string,
    notes: typeof b.notes === "string" ? b.notes : "",
  };
}

export async function POST(req: Request) {
  let parsed: unknown;
  try { parsed = await req.json(); } catch { return NextResponse.json({ error: "Invalid JSON" }, { status: 400 }); }
  const body = validate(parsed);
  if (!body) return NextResponse.json({ error: "Missing required fields" }, { status: 400 });

  const db = getDb();
  const mtnSnap = await db.collection("mountains").doc(body.mountainId).get();
  if (!mtnSnap.exists) return NextResponse.json({ error: "Mountain not found" }, { status: 404 });
  const mtn = mtnSnap.data() as { name: string; slug: string; nwacZoneId: string; snotelStationId: string };

  const ref = db.collection("projects").doc();
  const now = new Date().toISOString();
  const project: Project = {
    id: ref.id, name: body.name, mountainId: body.mountainId,
    mountainName: mtn.name, mountainSlug: mtn.slug,
    targetDateStart: body.targetDateStart, targetDateEnd: body.targetDateEnd,
    status: "active", notes: body.notes ?? "",
    createdAt: now, lastRefreshedAt: null, lastRefreshStatus: "pending",
  };
  await ref.set(project);

  // Side effects (spec §5): immediate refresh + worker fan-out + backfill.
  await publish("weather-refresh", { mountainId: body.mountainId, reason: "on_create" });
  await publish("nwac-refresh", { zoneId: mtn.nwacZoneId });
  await publish("snotel-refresh", { stationId: mtn.snotelStationId });
  await publish("backfill-refresh", { projectId: ref.id, mountainId: body.mountainId, targetDate: body.targetDateStart });

  return NextResponse.json(project, { status: 201 });
}
```

```ts
import { NextResponse } from "next/server";
import { getDb } from "@/lib/firebase-admin";

const CACHE = "public, max-age=300, stale-while-revalidate=600";
type Params = { params: Promise<{ id: string }> };
const PATCHABLE = ["name", "notes", "targetDateStart", "targetDateEnd"] as const;

export async function GET(_req: Request, { params }: Params) {
  const { id } = await params;
  const doc = await getDb().collection("projects").doc(id).get();
  if (!doc.exists) return NextResponse.json({ error: "Project not found" }, { status: 404 });
  return NextResponse.json({ id, ...(doc.data() as object) }, { headers: { "Cache-Control": CACHE } });
}

export async function PATCH(req: Request, { params }: Params) {
  const { id } = await params;
  const ref = getDb().collection("projects").doc(id);
  const doc = await ref.get();
  if (!doc.exists) return NextResponse.json({ error: "Project not found" }, { status: 404 });

  let body: unknown;
  try { body = await req.json(); } catch { return NextResponse.json({ error: "Invalid JSON" }, { status: 400 }); }
  const b = (body ?? {}) as Record<string, unknown>;
  const update: Record<string, unknown> = {};
  for (const key of PATCHABLE) if (typeof b[key] === "string") update[key] = b[key];
  if (Object.keys(update).length === 0) return NextResponse.json({ error: "No updatable fields" }, { status: 400 });

  await ref.update(update);
  return NextResponse.json({ id, ...(doc.data() as object), ...update });
}

export async function DELETE(_req: Request, { params }: Params) {
  const { id } = await params;
  const ref = getDb().collection("projects").doc(id);
  const doc = await ref.get();
  if (!doc.exists) return NextResponse.json({ error: "Project not found" }, { status: 404 });
  await ref.update({ status: "archived" });
  return NextResponse.json({ ok: true });
}
```

```ts
import { NextResponse } from "next/server";
import { getDb } from "@/lib/firebase-admin";
import { getBrowseRefreshMode } from "@/lib/env";
import { publish } from "@/lib/pubsub";

const CACHE = "public, max-age=300, stale-while-revalidate=600";
const STALE_MS = 3 * 60 * 60 * 1000; // ~3h (spec §4)
type Params = { params: Promise<{ slug: string }> };

function isStale(updatedAt: string | undefined): boolean {
  if (!updatedAt) return true;
  return Date.now() - new Date(updatedAt).getTime() > STALE_MS;
}

export async function GET(_req: Request, { params }: Params) {
  const { slug } = await params;
  const db = getDb();
  const mtnDoc = await db.collection("mountains").doc(slug).get();
  if (!mtnDoc.exists) return NextResponse.json({ error: "Mountain not found" }, { status: 404 });
  const mountain = { slug, ...(mtnDoc.data() as object) };

  const condDoc = await db.collection("mountainConditions").doc(slug).get();
  const conditions = condDoc.exists ? (condDoc.data() as { updatedAt?: string }) : null;

  // Get-or-refresh (spec §4): only in lazy mode, only when stale/absent.
  if (getBrowseRefreshMode() === "lazy" && isStale(conditions?.updatedAt)) {
    void publish("weather-refresh", { mountainId: slug, reason: "manual" }).catch(() => {});
  }

  return NextResponse.json({ mountain, conditions }, { headers: { "Cache-Control": CACHE } });
}
```

> Fire-and-forget: `void publish(...).catch(() => {})` so a publish failure never blocks serving stale/current data (spec §4). In tests `publish` resolves synchronously enough that `calls` is populated before the response is read; if a race appears, `await` the publish — the contract permits it (response still returns stale data).

```ts
import { NextResponse } from "next/server";
import { publish } from "@/lib/pubsub";

export async function POST(req: Request) {
  const mountainId = new URL(req.url).searchParams.get("mountainId");
  if (!mountainId) return NextResponse.json({ error: "mountainId is required" }, { status: 400 });
  await publish("weather-refresh", { mountainId, reason: "manual" });
  return NextResponse.json({ ok: true });
}
```
