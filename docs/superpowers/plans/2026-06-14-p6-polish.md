# P6 — Polish & POC Readiness — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Take the feature-complete UI from P4 (Dashboard, Create, calm-layer Detail, Browse, units toggle) and P5 (Freezing-Level hero, Confidence strip, Model Lab) and make it POC-demo-ready: responsive at every Cirque breakpoint, with loading / error / empty / pending states everywhere, last-refreshed + "updating…" indicators, share/copy-link, a verified units toggle, a real accessibility pass, an attribution footer, motion polish, a README, and a full coverage + UX re-check. **This is a polish phase** — most tasks modify an existing component to add a state or behavior and add a test asserting the new behavior; very little net-new structure.

**Architecture:** No new data sources or backend changes. All work lives in `app/**`, `components/**`, `lib/**`, `tests/e2e/**`, and `README.md`. Data continues to flow exactly as P3 established it (Route Handlers under `app/api/**`, served to client components via SWR; server-computed `tone`/`verdict` on `currentSummary`). Polish leans on the SWR hybrid pattern (`nextjs-patterns` skill): loading/error states come from SWR's `isLoading`/`error`; the "updating…" pill comes from `isValidating` + the API's get-or-refresh path (contract §4, §7). Units are read everywhere through `lib/units.ts` helpers (contract §12a). All visual decisions trace to `prototype-ui/prototype-design-review/project/DESIGN.md` (Cirque) — §14 motion, §15 responsive, §16 a11y, §20 (the open-questions items that become POC work here: empty/pending/off-season/stale states).

**Tech Stack:** Next.js 16.2.x (App Router, React 19.2), TypeScript, Tailwind + Cirque CSS-variable tokens in `app/globals.css`, hand-built SVG charts (no Recharts), SWR, Zustand (`lib/units.ts`), Vitest + Testing Library + `vitest-axe` (axe-core a11y assertions), Playwright (desktop 1280×800 + mobile iPhone 12, plus an explicit ≤680 viewport). Fonts Newsreader / Hanken Grotesk / IBM Plex Mono via `next/font/google`.

**Next.js 16 conventions** (apply throughout): dynamic `params`/`searchParams` are **async** (a `Promise` — `await` them in pages/handlers); GET Route Handlers are uncached by default (caching set via `Cache-Control`, contract §7); `serverExternalPackages` is top-level. P6 adds **no** new Route Handlers, so these only matter where a polish task touches an existing page/handler.
